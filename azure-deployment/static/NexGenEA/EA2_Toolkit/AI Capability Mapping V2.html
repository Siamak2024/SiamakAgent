Workshop Flow Guide

Step 1: Define AS-IS Capabilities

Document your organization's current capability landscape.

  • Manual Entry: Add capabilities one-by-one using the domain cards
  • Template Import: Use industry templates as a starting point
  • AI-Assisted APQC: Import capabilities from APQC framework with AI filtering
  • Lock Validated Capabilities: Click lock icon to mark capabilities as reviewed
Tip: Complete Step 1 when you have locked at least 60% of your AS-IS capabilities.

Step 2: Generate TO-BE Capabilities

Use AI to identify transformation opportunities and future-state capabilities.

  • Create TO-BE: AI generates future-state capabilities based on AS-IS and strategic context
  • Review Comparison: See side-by-side AS-IS vs TO-BE analysis
  • AI Insights: Automatic identification of gaps, duplicates, and transformation opportunities
  • Edit & Refine: Manually adjust TO-BE capabilities as needed
Note: TO-BE generation requires OpenAI API key and AS-IS capabilities.

Step 3: Review & Consolidate

Finalize your capability map and prepare for integration with downstream toolkits.

  • Consolidate: Merge AS-IS and TO-BE into a unified capability map
  • Export v2.0: Download capability assessment with full metadata
  • Store for Integration: Save to EA_DataManager for Gap Analysis toolkit
  • Verify Schema: Ensure export matches integration registry specification
Next: Use exported data in Gap Analysis and Transformation Roadmap toolkits.

Integration Governance

This toolkit follows strict integration principles to ensure seamless data flow:

  • Versioned exports (v2.0 schema with backward compatibility)
  • EA_DataManager API for integration storage
  • No breaking changes to existing toolkit chains
  • Documentation-first approach (see INTEGRATION_REGISTRY.md)
