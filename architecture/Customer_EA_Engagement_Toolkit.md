EA Engagement – Structured Implementation Instructions
Purpose
Implement an EA‑led, Agile, segment‑based engagement that:
Works across multiple customers
Supports Sales, Account Planning, and Executives
Focuses on Application Sunsetting & Modernization
Produces tangible, reviewable outputs incrementally
1. Engagement Setup – How to Start
Step 1: Confirm Engagement Context
Identify:
Segment (e.g. Insurance, Banking, Finance)
Number of customers (5–10)
Key customers (priority)
Define the common theme (e.g. application sunsetting / modernization)
✅ Output: Engagement context agreed
Step 2: Establish EA Mandate
Confirm EA‑led execution
Secure top‑management mandate
Position EA as strategic integrator, not documentation role
✅ Output: Clear mandate and authority to lead
2. Phase 0 – Initiation & Setup (Sprint 0)
Objective
Create alignment, governance, and execution readiness before any analysis.
Actions
Define engagement objectives and scope
Agree success criteria
Establish governance and decision forums
Create RACI matrix
Set Agile cadence (sprints, reviews)
Mandatory Deliverables
Engagement scope & objectives
RACI matrix
High‑level Agile plan (phases)
Initial stakeholder list
Tools
Engagement canvas
RACI template
Stakeholder map
Agile backlog (Epic level)
3. Agile Execution Model (Mandatory)
Structure
Epics = Engagement Phases
Stories = EA Outcomes
Each story produces a reviewable artifact
Epic Structure
E0 – Initiation & Setup
E1 – AS‑IS Assessment
E2 – White‑Spot & Opportunity Analysis
E3 – Customer Validation & Alignment
E4 – Target EA Vision
E5 – Roadmap & Enablement
✅ Rule: One story = one EA outcome
4. Phase 1 – AS‑IS Assessment (Sprint 1–2)
Objective
Understand current state fast, at the right level.
Actions
Run internal workshops with:
Customer Engagement team
CSM
Focus on:
Application landscape (lifecycle‑based)
Constraints
Known risks
Ongoing initiatives
Deliverables
AS‑IS architecture overview
Application lifecycle snapshot
Initial modernization / sunsetting candidates
Risk & constraint log
Tools
Application Portfolio Assessment (APA)
Lifecycle heatmap
Notes & actions log
5. Phase 2 – White‑Spot & Opportunity Analysis (Sprint 3)
Objective
Identify where value is missing, not just what exists.
Actions
Compare AS‑IS with:
Business drivers
Segment patterns
Identify:
Capability gaps
Redundancies
Technical debt hotspots
Deliverables
White‑spot map
Opportunity list
Prioritised themes
Tools
Capability map
White‑spot canvas
Value vs effort matrix
6. Phase 3 – Customer Validation & Alignment (Sprint 4–5)
Objective
Anchor EA work in real customer priorities.
Actions
Prepare and run customer workshops with:
CIO / Chief Architect
Business leader (where possible)
Validate:
Pain points
Priorities
Constraints
Deliverables
Confirmed priorities
Updated stakeholder map (top‑down)
Decision and assumption log
Tools
Workshop agenda template
Stakeholder mapping
Decision log
7. Phase 4 – Target EA Vision (Sprint 6–7)
Objective
Define a pragmatic future‑state EA vision.
Actions
Define:
Architecture principles
Target patterns
Modernization approach
Tailor per customer, reuse per segment
Deliverables
Target EA (Vivicta EA) overview
Architecture principles
Modernization & sunsetting candidate list
Tools
Target architecture views
Principle catalog
Reference architectures
8. Phase 5 – Roadmap & Enablement (Sprint 8)
Objective
Turn EA insight into action, sales value, and decisions.
Actions
Define phased roadmap:
Short / mid / long term
Link initiatives to:
Business value
Delivery options
Package executive & sales material
Deliverables
EA roadmap
Sales / executive narrative
Action & follow‑up list
Tools
Roadmap canvas
Initiative dependency map
Executive story template

AS‑IS Summary

Target EA Vision
Modernization & Sunsetting
Roadmap & Next Steps
Decisions, Risks & Assumptions
Actions & Follow‑Up
Appendices (notes, diagrams)
✅ This document is the single source of truth
10. Key Rules for Implementation (Non‑Negotiable)
EA leads — others contribute
No big‑bang documentation
Every phase delivers value
Reuse at segment level
Tailor at customer level
Optimize for decision‑making, not completeness
Final One‑Sentence Instruction
Implement EA engagements as an EA‑led, Agile, phase‑based initiative where each phase produces tangible, sales‑ and customer‑usable outputs, captured in a single structured engagement document and driven through an Epic‑to‑Story backlog.
